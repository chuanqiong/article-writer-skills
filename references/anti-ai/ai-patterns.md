# AI Sentence Patterns

## Alert Patterns

| Pattern | Risk Threshold | Rewrite Direction |
| --- | --- | --- |
| `不是...而是...` | 同文 `>= 2` 次 | 直接说结论，少做对仗 |
| `既...又...` | 同文 `>= 2` 次 | 拆成两句，补具体证据 |
| `通过...实现...` | 同文 `>= 3` 次 | 改成谁做了什么，结果是什么 |
| `一方面...另一方面...` | 同文 `>= 2` 次 | 直接列两点，不要模板句 |
| `不仅...还...` | 同文 `>= 2` 次 | 选择重点，不要机械并列 |
| `无论...都...` | 同文 `>= 1` 次 | 高风险，优先重写 |
| `从...到...` | 同文 `>= 3` 次 | 看是否真的需要范围表达 |
| `让...更...` | 同文 `>= 4` 次 | 改成更具体的动作或结果 |
| 因果链长句 | 连续多句 | 拆短句，留下一个明确因果 |

## Style Pass Heuristics

- 同一模式连用，比单次出现更危险。
- 对仗、平衡、完整，往往也是最像 AI 的地方。
- 如果一句话删掉连接结构后信息几乎没损失，说明这句太模板化。

## Better Rewrites

- 抽象结构 -> 具体动作
- 对仗句 -> 主次分明的两句
- 平衡句 -> 带立场的判断句
- 大而全总结 -> 一个明确结论
